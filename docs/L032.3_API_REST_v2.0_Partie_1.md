# L032.3_API_REST_v2.0.md --- Partie 1/2

# 1. Objet

Ce document définit les conventions de conception des endpoints REST de
TurfIA. Il décrit les contrats d'échange, les règles de validation et
les principes garantissant une API cohérente, stable et versionnée.

# 2. Structure d'un endpoint

Chaque endpoint comprend les éléments suivants :

-   URI versionnée ;
-   méthode HTTP ;
-   schéma d'entrée ;
-   schéma de sortie ;
-   codes HTTP normalisés ;
-   règles d'authentification ;
-   contraintes métier.

# 3. Cycle d'une requête

``` mermaid
sequenceDiagram
participant Client
participant API
participant Validation
participant Service
participant DB

Client->>API: HTTP Request
API->>Validation: Validation
Validation->>Service: Exécution
Service->>DB: Lecture / Écriture
DB-->>Service: Résultat
Service-->>API: Réponse
API-->>Client: JSON
```

# 4. Principes

Les contrats REST doivent rester indépendants de l'implémentation
interne afin de préserver la compatibilité des clients.

## 4.1 Contrat OpenAPI comme source de vérité

La documentation OpenAPI générée automatiquement (cf. L007 §2, L016
§11) constitue la source de vérité du contrat d'un endpoint : toute
divergence entre le comportement réel et le contrat publié est traitée
comme un défaut, jamais comme une mise à jour tacite acceptable de la
documentation.
