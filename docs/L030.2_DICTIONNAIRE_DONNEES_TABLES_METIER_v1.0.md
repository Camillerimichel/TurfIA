# L030.2 --- Dictionnaire de données --- Tables métier

## 1. Objet

Ce document spécifie les tables métier manipulées quotidiennement par
TurfIA.

------------------------------------------------------------------------

# 2. Table `reunion`

## Description

Une réunion regroupe l'ensemble des courses disputées sur un hippodrome
à une date donnée.

  Colonne         Type          Contraintes   Description
  --------------- ------------- ------------- ----------------------------
  id              BIGINT        PK            Identifiant
  date            DATE          NOT NULL      Date de réunion
  hippodrome_id   BIGINT        FK            Hippodrome
  numero          SMALLINT      NOT NULL      Numéro de réunion
  heure_debut     TIMESTAMP                   Début
  heure_fin       TIMESTAMP                   Fin
  statut          VARCHAR(20)                 Prévue, En cours, Terminée

### Contraintes

-   UNIQUE(date, hippodrome_id, numero)

### Index

-   idx_reunion_date
-   idx_reunion_hippodrome

------------------------------------------------------------------------

# 3. Table `course`

## Description

Une course appartient à une réunion.

  Colonne          Type            Contraintes
  ---------------- --------------- -------------
  id               BIGINT          PK
  reunion_id       BIGINT          FK
  numero           SMALLINT        NOT NULL
  nom              VARCHAR(150)    NOT NULL
  heure_depart     TIMESTAMP       INDEX
  discipline_id    BIGINT          FK
  type_course_id   BIGINT          FK
  distance_id      BIGINT          FK
  surface_id       BIGINT          FK
  etat_piste_id    BIGINT          FK
  allocation       DECIMAL(12,2)   
  nb_partants      SMALLINT        
  quinté           BOOLEAN         

### Contraintes

-   UNIQUE(reunion_id, numero)

### Relations

-   1 → N avec partant
-   1 → N avec analyse
-   1 → N avec resultat

------------------------------------------------------------------------

# 7. Table `partant`

## Description

Association entre une course et un cheval.

  Colonne         Type           Contraintes
  --------------- -------------- ---------------
  id              BIGINT         PK
  course_id       BIGINT         FK
  cheval_id       BIGINT         FK
  jockey_id       BIGINT         FK
  entraineur_id   BIGINT         FK
  numero          SMALLINT       NOT NULL
  corde           SMALLINT       
  poids           DECIMAL(5,1)   
  valeur          DECIMAL(5,2)   
  age             SMALLINT       
  ferrure         VARCHAR(30)    
  musique         VARCHAR(80)    
  non_partant     BOOLEAN        DEFAULT FALSE

### Contraintes

-   UNIQUE(course_id, numero)
-   UNIQUE(course_id, cheval_id)

### Index

-   idx_partant_course
-   idx_partant_cheval
-   idx_partant_jockey
-   idx_partant_entraineur

------------------------------------------------------------------------

# 8. Table `cote`

## Description

Historique des cotes.

  Colonne      Type
  ------------ --------------
  id           BIGINT
  partant_id   BIGINT
  operateur    VARCHAR(40)
  cote         DECIMAL(6,2)
  evolution    DECIMAL(6,2)
  date_maj     TIMESTAMP

### Index

-   idx_cote_partant
-   idx_cote_date

Une même journée peut contenir plusieurs milliers de mises à jour de
cotes.

# 10. Flux métier

``` text
Réunion
   │
   └── Course
          │
          ├── Partants
          │      ├── Cheval
          │      ├── Jockey
          │      └── Entraîneur
          │
          ├── Cotes
          └── Résultats
```

------------------------------------------------------------------------

# 11. Règles métier

-   une course appartient à une seule réunion ;
-   un cheval peut participer à plusieurs courses ;
-   un partant ne peut apparaître qu'une fois dans une course ;
-   une cote est historisée et jamais remplacée ;
-   les résultats officiels sont figés après validation.

------------------------------------------------------------------------

# 12. Historisation

Les tables métier ne font l'objet d'aucune suppression physique.

Toute évolution significative est historisée afin de garantir la
reproductibilité des analyses TurfIA.
