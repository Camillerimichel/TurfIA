# L032.2_API_REST_v1.0

# API REST — Ressources métier

Version : 1.0

## 1. Objet

Ce document décrit l'organisation fonctionnelle des ressources exposées par l'API REST de TurfIA. Il complète la vue d'ensemble présentée dans L007 et sert de référence pour l'implémentation FastAPI.

## 2. Principes

- Architecture REST.
- Format JSON unique.
- Endpoints versionnés.
- Séparation entre lecture et écriture.
- Validation systématique des entrées.

## 3. Domaines d'API

| Domaine | Ressources principales |
|---------|------------------------|
| Référentiels | chevaux, jockeys, entraîneurs, hippodromes |
| Courses | réunions, courses, partants |
| Analyses | pré-analyses, analyses finales, scores |
| Paris | propositions, mises, simulations |
| Résultats | arrivées, rapports, ROI |
| Administration | paramètres, tâches, journaux |

## 4. Convention d'URL

```
/api/v1/{domaine}/{ressource}
```

Les opérations CRUD utilisent les méthodes HTTP standards.

## 5. Contrats

Chaque ressource définit :

- schémas Pydantic ;
- règles de validation ;
- codes de retour HTTP ;
- pagination lorsque nécessaire ;
- gestion normalisée des erreurs.

## 6. Sécurité

- Authentification centralisée.
- Autorisation par rôle.
- Journalisation des appels.
- Validation des droits avant exécution.
