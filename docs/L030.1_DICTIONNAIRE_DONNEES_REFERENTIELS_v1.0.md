## 1. Objet

Cette première partie décrit les tables de référence utilisées par
TurfIA. Elles sont communes à l'ensemble des traitements et évoluent peu
dans le temps.

## 2. Principes

-   Clé primaire : `id`
-   Clés étrangères : `id_<table>`
-   Toutes les tables comportent les champs techniques :
    -   `date_creation TIMESTAMP`
    -   `date_modification TIMESTAMP`
    -   `actif BOOLEAN`

------------------------------------------------------------------------

# 3. Table `hippodrome`

## Description

Référentiel des hippodromes.

  Colonne     Type           Contraintes   Description
  ----------- -------------- ------------- ------------------
  id          BIGINT         PK            Identifiant
  nom         VARCHAR(100)   UNIQUE        Nom officiel
  ville       VARCHAR(80)                  Ville
  pays        VARCHAR(50)                  Pays
  corde_id    BIGINT         FK            Sens de la corde
  altitude    INTEGER                      Altitude
  latitude    DECIMAL(9,6)                 Latitude
  longitude   DECIMAL(9,6)                 Longitude

### Index

-   idx_hippodrome_nom

### Relations

-   1 → N avec `reunion`

# 10. Contraintes communes

Toutes les tables de référence :

-   sont alimentées par les imports automatiques ;
-   sont historisées si leur contenu évolue ;
-   ne sont jamais supprimées physiquement ;
-   sont utilisées exclusivement via des clés étrangères.

------------------------------------------------------------------------

# 11. Index communs

Chaque table possède :

-   une clé primaire ;
-   un index sur le libellé ;
-   les index nécessaires aux recherches fréquentes.

------------------------------------------------------------------------

# 12. Évolutivité

De nouveaux référentiels pourront être ajoutés sans modifier les tables
métier.

Exemples :

-   catégories PMU ;
-   types de ferrure ;
-   équipements ;
-   conditions météorologiques enrichies ;
-   familles de paris.

------------------------------------------------------------------------

## 13. Droits d'accès

Les tables de référence sont accessibles en lecture au rôle applicatif
`turfia_readonly` (cf. L011 §16.5) ; leur écriture est réservée au
processus de collecte (cf. L009, L010) et à l'administration
fonctionnelle (cf. L025 §3.2).
