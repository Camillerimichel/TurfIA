# L032.2_API_REST_v2.0.md --- Partie 1/2

# 1. Objet

Ce document décrit l'organisation fonctionnelle des ressources exposées
par l'API REST de TurfIA. Il complète la vue d'ensemble présentée dans
L007 en définissant les domaines métier, les conventions d'exposition
des ressources et les responsabilités associées.

# 2. Principes

Les ressources REST sont organisées selon les principes suivants :

-   regroupement par domaine métier ;
-   séparation entre consultation et modification ;
-   contrats d'échange stables ;
-   validation systématique des données ;
-   indépendance vis-à-vis de l'interface cliente.

# 3. Domaines fonctionnels

  Domaine          Ressources principales
  ---------------- --------------------------------------------
  Référentiels     chevaux, jockeys, entraîneurs, hippodromes
  Courses          réunions, courses, partants
  Analyses         pré-analyses, analyses finales
  Résultats        arrivées, ROI, statistiques
  Administration   paramètres, tâches, journaux

``` mermaid
flowchart LR
Client --> API
API --> Referentiels
API --> Courses
API --> Analyses

```

# 4. Responsabilités

Chaque domaine expose exclusivement les opérations relevant de son
périmètre métier afin de limiter le couplage entre services.
