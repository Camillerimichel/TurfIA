# 5. Contraintes d'exposition

Les ressources REST doivent respecter les contraintes suivantes :

-   identifiants techniques stables ;
-   pagination homogène ;
-   filtrage normalisé ;
-   validation des entrées ;
-   codes HTTP cohérents ;
-   messages d'erreur standardisés.

Les évolutions incompatibles sont gérées par versionnement afin de
préserver les consommateurs existants.
