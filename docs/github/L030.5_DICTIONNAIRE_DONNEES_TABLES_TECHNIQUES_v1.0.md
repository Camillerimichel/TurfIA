# L030.5 --- Dictionnaire de données --- Tables techniques

## 1. Objet

Les tables techniques assurent le fonctionnement interne de TurfIA.
Elles ne portent aucune donnée hippique mais garantissent la
configuration, la traçabilité, les migrations, les tâches planifiées et
la supervision.

------------------------------------------------------------------------

# 2. Vue d'ensemble

``` text
Configuration
│
├── parametre
├── version
├── migration
├── journal
├── tache
├── utilisateur
├── role
└── audit
```

------------------------------------------------------------------------

# 3. Table `parametre`

## Description

Contient les paramètres fonctionnels modifiables sans changement du
code.

  Colonne             Type           Contraintes   Description
  ------------------- -------------- ------------- -----------------------------------
  id                  BIGINT         PK            Identifiant
  cle                 VARCHAR(120)   UNIQUE        Nom du paramètre
  valeur              TEXT           NOT NULL      Valeur
  type                VARCHAR(20)                  Integer, Decimal, Boolean, String
  description         TEXT                         Documentation
  date_modification   TIMESTAMP                    Dernière modification

Exemples :

-   score_minimum
-   budget_reference
-   ponderation_consensus
-   ponderation_cotes

------------------------------------------------------------------------

# 4. Table `version`

  Colonne            Type
  ------------------ -------------
  id                 BIGINT
  version            VARCHAR(20)
  commit_git         VARCHAR(80)
  branche            VARCHAR(40)
  date_publication   TIMESTAMP
  commentaire        TEXT

Permet d'identifier précisément la version ayant produit une analyse.

------------------------------------------------------------------------

# 5. Table `migration`

  Colonne    Type
  ---------- --------------
  id         BIGINT
  version    VARCHAR(40)
  fichier    VARCHAR(255)
  checksum   VARCHAR(128)
  debut      TIMESTAMP
  fin        TIMESTAMP
  duree_ms   INTEGER
  resultat   VARCHAR(20)

Une migration n'est exécutée qu'une seule fois.

------------------------------------------------------------------------

# 6. Table `journal`

## Description

Journal technique de l'application.

  Colonne          Type
  ---------------- -------------
  id               BIGINT
  date_evenement   TIMESTAMP
  niveau           VARCHAR(20)
  composant        VARCHAR(60)
  correlation_id   VARCHAR(64)
  message          TEXT
  exception        TEXT

### Valeurs possibles

-   DEBUG
-   INFO
-   WARNING
-   ERROR
-   CRITICAL

------------------------------------------------------------------------

# 7. Table `tache`

## Description

Historique des traitements automatiques.

  Colonne       Type
  ------------- --------------
  id            BIGINT
  nom           VARCHAR(100)
  categorie     VARCHAR(40)
  debut         TIMESTAMP
  fin           TIMESTAMP
  duree_ms      INTEGER
  statut        VARCHAR(20)
  commentaire   TEXT

Exemples :

-   Pré-analyse
-   Analyse finale
-   Contrôle ROI
-   Sauvegarde
-   Statistiques

------------------------------------------------------------------------

# 8. Table `utilisateur`

  Colonne              Type
  -------------------- --------------
  id                   BIGINT
  login                VARCHAR(80)
  mot_de_passe         VARCHAR(255)
  nom                  VARCHAR(120)
  email                VARCHAR(150)
  role_id              BIGINT
  actif                BOOLEAN
  derniere_connexion   TIMESTAMP

Les mots de passe sont stockés exclusivement sous forme hachée.

------------------------------------------------------------------------

# 9. Table `role`

  Colonne       Type
  ------------- -------------
  id            BIGINT
  nom           VARCHAR(40)
  description   TEXT

Valeurs recommandées :

-   Administrateur
-   Analyste
-   Consultation
-   Automatisation

------------------------------------------------------------------------

# 10. Table `audit`

## Description

Traçabilité des opérations sensibles.

  Colonne          Type
  ---------------- --------------
  id               BIGINT
  utilisateur_id   BIGINT
  date_action      TIMESTAMP
  action           VARCHAR(120)
  objet            VARCHAR(120)
  ancien_etat      TEXT
  nouvel_etat      TEXT

Cette table répond aux besoins d'audit et de conformité.

------------------------------------------------------------------------

# 11. Contraintes communes

Toutes les tables techniques :

-   possèdent une clé primaire ;
-   sont historisées ;
-   sont sauvegardées quotidiennement ;
-   sont exclues des suppressions automatiques.

------------------------------------------------------------------------

# 12. Exploitation

Ces tables alimentent :

-   l'administration ;
-   la supervision ;
-   les journaux ;
-   les tableaux de bord techniques ;
-   les procédures de maintenance ;
-   les audits de sécurité.

Elles constituent le socle opérationnel de TurfIA.
