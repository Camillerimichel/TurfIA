# 5. Gestion des erreurs

Toutes les erreurs retournées par l'API utilisent une structure homogène
comprenant :

-   un code HTTP ;
-   un code fonctionnel ;
-   un message explicite ;
-   des informations de diagnostic lorsque cela est autorisé.

Les erreurs sont journalisées afin de faciliter les investigations et la
supervision.

# 6. Compatibilité et versionnement

Les évolutions compatibles sont introduites sans rupture de contrat.
Toute modification incompatible entraîne la création d'une nouvelle
version d'API.

Les contrats publiés constituent un engagement vis-à-vis des
consommateurs et ne doivent pas dépendre des choix d'implémentation
internes.
