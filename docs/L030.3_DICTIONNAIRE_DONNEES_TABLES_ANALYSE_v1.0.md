## 1. Objet

Les tables d'analyse conservent toutes les exécutions du moteur TurfIA
afin de garantir la reproductibilité des décisions et le calcul
historique du ROI.

# 3. Table `analyse`

## Description

Une analyse correspond à une exécution complète de TurfIA sur une
course.

  Colonne           Type           Contraintes   Description
  ----------------- -------------- ------------- ----------------------
  id                BIGINT         PK            Identifiant
  course_id         BIGINT         FK            Course
  version           SMALLINT       NOT NULL      1=Pré,2=Finale
  date_calcul       TIMESTAMP      NOT NULL      Horodatage
  score_confiance   DECIMAL(5,2)   CHECK 0-100   Score TurfIA
  risque            DECIMAL(5,2)                 Niveau de risque
  roi_theorique     DECIMAL(8,2)                 ROI estimé
  decision          VARCHAR(20)                  Jouer / Ne pas jouer
  budget            DECIMAL(8,2)                 Budget recommandé
  commentaire       TEXT                         Synthèse

### Contraintes

-   UNIQUE(course_id, version)

### Index

-   idx_analyse_course
-   idx_analyse_score
-   idx_analyse_date

------------------------------------------------------------------------

# 4. Table `analyse_partant`

## Description

Résultat détaillé du calcul pour chaque cheval.

  Colonne          Type
  ---------------- --------------
  id               BIGINT
  analyse_id       BIGINT
  partant_id       BIGINT
  score            DECIMAL(6,2)
  rang             SMALLINT
  consensus        DECIMAL(5,2)
  evolution_cote   DECIMAL(6,2)
  value_bet        BOOLEAN
  confiance        DECIMAL(5,2)
  commentaire      TEXT

### Contraintes

-   UNIQUE(analyse_id, partant_id)

------------------------------------------------------------------------

# 5. Table `selection`

## Description

Classement fonctionnel des chevaux retenus.

  Colonne           Type
  ----------------- -------------
  id                BIGINT
  analyse_id        BIGINT
  partant_id        BIGINT
  categorie         VARCHAR(30)
  ordre_affichage   SMALLINT

### Valeurs possibles

-   Base
-   Chance régulière
-   Outsider
-   Tocard
-   Écarté

------------------------------------------------------------------------

# 6. Table `pari`

## Description

Paris proposés automatiquement.

  Colonne       Type
  ------------- ---------------
  id            BIGINT
  analyse_id    BIGINT
  type_pari     VARCHAR(40)
  combinaison   VARCHAR(255)
  mise          DECIMAL(8,2)
  gain_estime   DECIMAL(10,2)
  roi_estime    DECIMAL(8,2)

------------------------------------------------------------------------

# 7. Table `controle_roi`

## Description

Résultat réel de l'analyse après publication de l'arrivée officielle.

  Colonne       Type
  ------------- ---------------
  id            BIGINT
  analyse_id    BIGINT
  mise          DECIMAL(8,2)
  gains         DECIMAL(10,2)
  profit        DECIMAL(10,2)
  roi           DECIMAL(8,2)
  valide        BOOLEAN
  commentaire   TEXT

------------------------------------------------------------------------

# 8. Règles métier

-   une course possède plusieurs versions d'analyse ;
-   une analyse référence tous les partants ;
-   les sélections sont dérivées des scores ;
-   le contrôle ROI n'est créé qu'après le résultat officiel ;
-   aucune analyse n'est modifiée après validation.

------------------------------------------------------------------------

# 9. Historisation

Toutes les versions sont conservées.

Les recalculs créent une nouvelle version sans remplacer les
précédentes.

------------------------------------------------------------------------

# 10. Exploitation

Ces tables alimentent :

-   les tableaux de bord ;
-   les statistiques ;
-   les comparaisons de versions ;
-   l'amélioration continue du modèle ;
-   les rapports quotidiens.
