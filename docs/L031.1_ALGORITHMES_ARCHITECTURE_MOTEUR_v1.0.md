## 1. Objectif

Ce document spécifie le fonctionnement du moteur décisionnel de TurfIA.

Il constitue la référence unique pour l'implémentation des algorithmes
de sélection.

------------------------------------------------------------------------

# 2. Principes

Le moteur poursuit deux objectifs :

1.  Décider si une course doit être jouée.
2.  Sélectionner les chevaux uniquement si la course est jugée
    favorable.

La qualité de la course est prioritaire sur la qualité des chevaux.

------------------------------------------------------------------------

# 3. Chaîne de calcul

``` text
Import des données
        │
        ▼
Contrôle qualité
        │
        ▼
Pré-analyse
        │
        ▼
Calcul des critères
        │
        ▼
Calcul Score TurfIA
        │
        ▼
Calcul du risque
        │
        ▼
Calcul ROI théorique
        │
        ▼
Décision Jouer / Ne pas jouer
        │
        ▼
Classement des chevaux
        │
        ▼
Construction des paris
```

------------------------------------------------------------------------

# 4. Variables d'entrée

Le moteur exploite notamment :

-   caractéristiques de la course ;
-   historique hippodrome ;
-   discipline ;
-   état de la piste ;
-   distance ;
-   musique des chevaux ;
-   statistiques jockey ;
-   statistiques entraîneur ;
-   consensus presse ;
-   évolution des cotes ;
-   historiques TurfIA.

Toutes les données sont horodatées.

------------------------------------------------------------------------

# 5. Familles de critères

Le score TurfIA est construit à partir de plusieurs familles
indépendantes.

  Famille        Exemple
  -------------- ---------------------
  Marché         Évolution des cotes
  Presse         Consensus
  Forme          Musique
  Aptitude       Distance, terrain
  Statistiques   Jockey, entraîneur
  Historique     Hippodrome
  Valeur         Value Bet
  Régularité     Historique récent

Chaque famille possède une pondération configurable.

------------------------------------------------------------------------

# 6. Cycle de calcul

Chaque course suit systématiquement les étapes suivantes :

1.  Validation des données.
2.  Calcul des indicateurs.
3.  Normalisation.
4.  Pondération.
5.  Calcul du score.
6.  Évaluation du risque.
7.  Estimation du ROI.
8.  Décision.
9.  Sélection.
10. Archivage.

------------------------------------------------------------------------

# 7. Normalisation

Tous les indicateurs sont convertis sur une échelle commune comprise
entre 0 et 100.

Cela permet de comparer des variables de nature différente.

------------------------------------------------------------------------

# 8. Score TurfIA

Le score final est défini par :

``` text
Score TurfIA

=

Σ (Critère × Pondération)
```

Les pondérations sont stockées en base de données.

Aucune pondération n'est codée en dur.

------------------------------------------------------------------------

# 9. Décision

Le score produit quatre niveaux :

  Score    Décision
  -------- -------------------
  \< 60    Ne pas jouer
  60--74   Jeu prudent
  75--84   Jeu normal
  ≥ 85     Forte opportunité

Ces seuils sont paramétrables.

------------------------------------------------------------------------

# 10. Traçabilité

Chaque calcul conserve :

-   les critères utilisés ;
-   les pondérations ;
-   la version du modèle ;
-   les résultats intermédiaires ;
-   le score final.

Chaque décision est ainsi parfaitement reproductible.

------------------------------------------------------------------------

# 11. Évolutivité

L'ajout d'un nouveau critère consiste uniquement à :

1.  créer l'indicateur ;
2.  définir sa normalisation ;
3.  définir sa pondération ;
4.  l'intégrer au calcul global.

Le reste du moteur demeure inchangé.

------------------------------------------------------------------------

## 12. Validation des évolutions

Conformément à L006 ADR-004 (absence d'apprentissage en ligne), toute
modification de ce moteur est un changement de version explicite,
validé par rejeu complet de l'historique (cf. L020 §8.3) avant
adoption. Le détail de la méthodologie de validation est spécifié en
L031.7.
