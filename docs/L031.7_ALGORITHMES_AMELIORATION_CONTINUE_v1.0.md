# L031.7 --- Algorithmes TurfIA --- Amélioration continue du modèle

## 1. Objectif

L'amélioration continue garantit que le modèle TurfIA évolue uniquement
lorsqu'une évolution apporte un gain mesurable sur les performances
historiques.

Aucune modification n'est validée sur une intuition ou sur une seule
course.

------------------------------------------------------------------------

## 2. Principes

Les évolutions reposent sur :

-   des mesures objectives ;
-   un historique complet des analyses ;
-   des tests comparatifs ;
-   une validation statistique.

Le modèle reste stable tant qu'une nouvelle version n'a pas démontré sa
supériorité.

------------------------------------------------------------------------

## 3. Sources d'amélioration

Les axes d'amélioration comprennent notamment :

-   pondérations des critères ;
-   nouvelles métriques ;
-   évolution des cotes ;
-   consensus presse ;
-   statistiques jockey ;
-   statistiques entraîneur ;
-   historique des hippodromes ;
-   détection des Value Bets.

Chaque évolution est isolée afin de mesurer précisément son impact.

------------------------------------------------------------------------

## 4. Méthodologie

Pour chaque évolution :

1.  créer une nouvelle version du modèle ;
2.  exécuter les analyses sur un historique identique ;
3.  comparer les résultats ;
4.  mesurer les écarts ;
5.  décider de l'adoption ou du rejet.

Les comparaisons sont réalisées à budget constant.

------------------------------------------------------------------------

## 5. Indicateurs suivis

Les principaux indicateurs sont :

  Indicateur             Description
  ---------------------- --------------------------------
  ROI global             Rentabilité cumulée
  ROI par score          Performance selon la confiance
  ROI par hippodrome     Performance locale
  ROI par type de pari   Qualité des recommandations
  Taux de réussite       Courses gagnantes
  Drawdown               Perte maximale
  Stabilité              Variabilité des résultats

------------------------------------------------------------------------

## 6. Validation

Une évolution est validée uniquement si :

-   le ROI progresse de manière significative ;
-   le drawdown n'augmente pas de façon excessive ;
-   les performances restent stables sur plusieurs périodes ;
-   aucune régression majeure n'est observée.

Dans le cas contraire, la version est rejetée.

------------------------------------------------------------------------

## 7. Versionnement

Chaque évolution reçoit :

-   un identifiant de version ;
-   une date ;
-   un commit Git ;
-   une documentation ;
-   les paramètres utilisés.

Les anciennes versions restent disponibles pour comparaison.

------------------------------------------------------------------------

## 8. Pseudo-code

``` text
charger historique

pour chaque version

    recalculer toutes les analyses

    calculer les statistiques

comparer les versions

si amélioration validée

    promouvoir la nouvelle version

sinon

    conserver la version actuelle
```

------------------------------------------------------------------------

## 9. Traçabilité

Chaque campagne d'évaluation conserve :

-   la version comparée ;
-   les paramètres ;
-   les résultats détaillés ;
-   les graphiques de comparaison ;
-   la décision finale.

Cette traçabilité permet de justifier toute évolution du moteur.

------------------------------------------------------------------------

## 10. Conclusion

L'amélioration continue constitue un principe fondateur de TurfIA.

Le moteur évolue progressivement grâce aux résultats observés, tout en
garantissant la reproductibilité des analyses et la comparabilité des
performances historiques.

Ce document clôt la spécification des algorithmes TurfIA et sert de
référence pour toute évolution future du modèle décisionnel.
