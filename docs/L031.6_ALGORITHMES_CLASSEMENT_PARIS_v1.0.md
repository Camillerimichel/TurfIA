# L031.6 --- Algorithmes TurfIA --- Classement final et génération des paris

## 0. Métadonnées du document

| Champ | Valeur |
| --- | --- |

| Version | 1.0 |

## 1. Objectif

Ce document décrit les règles permettant de transformer les scores
calculés par TurfIA en recommandations opérationnelles.

Le classement final constitue la dernière étape avant la génération
automatique des paris.

------------------------------------------------------------------------

## 2. Données d'entrée

Le classement utilise les indicateurs suivants :

-   Score TurfIA
-   Niveau de risque
-   ROI théorique
-   Score Value Bet
-   Consensus presse
-   Évolution des cotes
-   Aptitude au parcours
-   Statistiques jockey
-   Statistiques entraîneur

Tous ces indicateurs sont déjà calculés par les étapes précédentes.

------------------------------------------------------------------------

## 3. Calcul du classement

Les chevaux sont triés selon :

``` text
Score Final

=

Score TurfIA

+

Bonus Value Bet

-

Malus Risque
```

Les coefficients exacts sont stockés dans la table `parametre`.

En cas d'égalité :

1.  meilleur Score TurfIA ;
2.  meilleur ROI théorique ;
3.  meilleur consensus ;
4.  meilleure évolution des cotes.

------------------------------------------------------------------------

## 4. Catégorisation

Chaque cheval reçoit une catégorie fonctionnelle.

  Catégorie          Critères généraux
  ------------------ ------------------------------------------
  Base               Score très élevé et risque faible
  Chance régulière   Forte probabilité de figurer à l'arrivée
  Outsider           Probabilité moyenne mais ROI intéressant
  Tocard             Forte cote avec potentiel identifié
  Écarté             Faible score ou risque excessif

Ces catégories sont recalculées à chaque analyse.

------------------------------------------------------------------------

## 5. Construction des paris

Les paris sont générés automatiquement à partir des catégories.

### Simple Gagnant

-   Base n°1.

### Simple Placé

-   Base n°1.
-   Chance régulière n°1.

### Couplé Gagnant

-   Base n°1 + Base n°2.

### Couplé Placé

-   Base n°1 + Chance régulière n°1.

### 2 sur 4

-   Deux Bases.
-   Deux Chances régulières.

### Quinté Flexi

Combinaison automatique :

-   Bases ;
-   Chances régulières ;
-   Outsider principal ;
-   Tocard éventuel si le ROI attendu reste positif.

------------------------------------------------------------------------

## 6. Gestion du budget

Le budget dépend exclusivement du score de confiance de la course.

  Score    Budget
  -------- --------
  \<60     0 €
  60--74   10 €
  75--84   25 €
  ≥85      50 €

Aucune augmentation de mise n'est autorisée après une perte (interdiction
explicite de toute logique de martingale).

------------------------------------------------------------------------

## 7. Optimisation

Avant validation, chaque pari est contrôlé :

-   cohérence avec le score ;
-   cohérence avec le ROI ;
-   niveau de risque ;
-   redondance éventuelle.

Les paris dont l'espérance est négative sont supprimés.

------------------------------------------------------------------------

## 8. Pseudo-code

``` text
trier les chevaux

attribuer une catégorie

construire les paris

calculer le budget

contrôler la cohérence

enregistrer la sélection
```

------------------------------------------------------------------------

## 9. Traçabilité

Chaque pari conserve :

-   les chevaux sélectionnés ;
-   les scores utilisés ;
-   la version du modèle ;
-   le budget conseillé ;
-   le ROI théorique.

Ces informations permettent la comparaison avec les résultats réels.

------------------------------------------------------------------------

## 10. Validation

La qualité des sélections est mesurée quotidiennement par :

-   taux de réussite des Bases ;
-   taux de réussite des Outsiders ;
-   ROI des paris ;
-   stabilité des recommandations.

Les résultats alimentent directement le processus d'amélioration
continue de TurfIA.
