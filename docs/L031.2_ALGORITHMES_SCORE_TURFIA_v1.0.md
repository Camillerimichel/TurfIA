## 1. Objectif

Ce document spécifie le calcul du Score TurfIA. Il formalise les
indicateurs, leur normalisation et leur agrégation.

------------------------------------------------------------------------

## 2. Principe général

Le score est obtenu par une somme pondérée de familles de critères.

``` text
Score = Σ (Score_normalisé_i × Pondération_i)
```

Les pondérations sont lues dans la table `parametre` et non codées en
dur.

------------------------------------------------------------------------

## 3. Familles de critères

  Famille          Exemple d'indicateurs
  ---------------- ----------------------------------
  Marché           évolution des cotes, stabilité
  Presse           consensus, taux de citation
  Forme            musique, régularité
  Aptitude         distance, terrain, hippodrome
  Professionnels   statistiques jockey / entraîneur
  Historique       performances TurfIA passées
  Value            écart probabilité / cote
  Contexte         nombre de partants, homogénéité

Chaque famille produit un sous-score de 0 à 100.

------------------------------------------------------------------------

## 4. Normalisation

Tous les indicateurs sont convertis sur une échelle commune.

Exemple linéaire :

``` text
Score = 100 × (Valeur - Min) / (Max - Min)
```

Pour les critères où une faible valeur est favorable :

``` text
Score = 100 - Score_normalisé
```

Les bornes sont paramétrables.

------------------------------------------------------------------------

## 5. Calcul des sous-scores

### Marché

Variables :

-   évolution des cotes ;
-   dispersion des favoris ;
-   stabilité du marché.

### Presse

Variables :

-   consensus ;
-   nombre de citations ;
-   convergence des pronostics.

### Forme

Variables :

-   musique récente ;
-   régularité ;
-   absence.

### Aptitude

Variables :

-   distance ;
-   surface ;
-   terrain ;
-   hippodrome.

### Professionnels

Variables :

-   réussite jockey ;
-   réussite entraîneur ;
-   réussite du couple.

------------------------------------------------------------------------

## 6. Agrégation

Le score global est calculé par :

``` text
Score_global =
Σ(SousScore_i × Poids_i)
/ Σ(Poids_i)
```

Les poids peuvent être modifiés sans changer l'algorithme.

### 6.1 Familles sans donnée exploitable (clarification d'implémentation, 2026-07-10)

Le SAD ne précise pas comment traiter une famille dont la donnée sous-jacente
est absente ou statistiquement trop peu fiable (ex. musique inconnue,
échantillon jockey/entraîneur/hippodrome inférieur à 3 courses). Décision
actée avec l'utilisateur après un vrai bug constaté (score systématiquement
plafonné pour la quasi-totalité des courses, cf. PROJECT_STATE.md) :

- Une famille sans donnée exploitable est **exclue** de `Σ(SousScore_i ×
  Poids_i) / Σ(Poids_i)` — elle ne participe ni au numérateur ni au
  dénominateur.
- Elle n'est **jamais** comptée à un score neutre (50) à plein poids : cela
  reviendrait à affirmer une confiance moyenne sur une donnée qui n'existe
  pas, et dilue mécaniquement le score dès qu'une ou plusieurs familles
  manquent de données — précisément la situation la plus fréquente en début
  de vie du moteur (peu d'hippodromes/jockeys/entraîneurs ont encore 3
  courses réelles contrôlées).
- Seules Marché et (si le classement presse a pu être obtenu) Presse sont
  quasi certaines d'être présentes ; Forme, Professionnels, Historique et
  Aptitude peuvent chacune être absentes indépendamment les unes des autres.
- Implémentation : `src/algorithms/indicateurs.py` (chaque fonction de
  sous-score retourne `None` plutôt qu'un score neutre quand la donnée est
  insuffisante) et `src/services/preparation_service.py` (n'ajoute une
  famille à `sous_scores` que si sa valeur n'est pas `None`).
- Ceci ne change ni les pondérations (restent à 1,0 chacune par défaut, cf.
  §10 : toute pondération doit être validée sur historique avant adoption,
  prématuré tant que peu de données réelles sont disponibles) ni les seuils
  de décision (§7 ci-dessous, inchangés).

## 8. Pseudo-code

``` text
pour chaque course

    calculer tous les indicateurs

    normaliser

    calculer chaque sous-score

    appliquer les pondérations

    calculer le score global

    enregistrer tous les résultats
```

------------------------------------------------------------------------

## 9. Traçabilité

Chaque calcul enregistre :

-   version du modèle ;
-   paramètres utilisés ;
-   sous-scores ;
-   score final ;
-   date de calcul.

------------------------------------------------------------------------

## 10. Validation

Toute modification d'une pondération ou d'une formule doit être comparée
à l'historique TurfIA avant adoption.

Aucune évolution ne peut être validée sans amélioration statistiquement
mesurable du ROI.

### 10.1 Prévention du surapprentissage

Toute nouvelle pondération est validée hors échantillon (sur une
période distincte de celle ayant servi à son calibrage) afin d'éviter
un surapprentissage sur l'historique disponible (cf. L002 §8.1 risques
pour la vision, L031.7 §4).
