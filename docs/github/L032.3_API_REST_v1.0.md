# L032.3_API_REST_v1.0

# API REST — Contrats des endpoints

Version : 1.0

## 1. Objet

Ce document définit les conventions de conception des endpoints REST de TurfIA. Il constitue le cadre de référence pour l'implémentation FastAPI, sans détailler chaque route individuellement.

## 2. Structure d'un endpoint

Chaque endpoint est défini par :

| Élément | Description |
|---------|-------------|
| URI | Ressource versionnée |
| Méthode HTTP | GET, POST, PUT, PATCH ou DELETE |
| Schéma d'entrée | Modèle Pydantic validé |
| Schéma de sortie | Réponse JSON normalisée |
| Codes HTTP | Réponses standardisées |
| Authentification | Selon le niveau d'accès requis |

## 3. Conventions

- Préfixe commun : `/api/v1`
- Noms de ressources au pluriel.
- Identifiants techniques dans le chemin.
- Filtres via paramètres de requête.
- Pagination pour les collections.
- Tri et filtrage homogènes.

## 4. Réponses

Toutes les réponses respectent une structure cohérente comprenant :

- données retournées ;
- métadonnées éventuelles (pagination, total, version) ;
- messages d'erreur normalisés si nécessaire.

## 5. Gestion des erreurs

Les erreurs sont classées par catégories :

- validation ;
- authentification ;
- autorisation ;
- ressource introuvable ;
- conflit métier ;
- erreur interne.

Chaque erreur possède un code HTTP et un message explicite.

## 6. Compatibilité

Les évolutions de l'API doivent préserver la compatibilité des clients par versionnement explicite et dépréciation progressive.
